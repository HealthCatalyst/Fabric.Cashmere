Typography Guidelines

Last updated January 21, 2021
Overview

For web developers, the following type properties and classes are defined in the Cashmere library's typography.scss stylesheet. Refer to the Getting Started developer guide for instructions on importing these styles into your project.

Ellipsis

For truncating a single line of text based on the width of it's containing div, use .hc-text-ellipsis, which is a helper class for the following commonly repeated css: 


    overflow: hidden;
    text-overflow: ellipsis;
    white-space: nowrap;
An example of ellipsis in action

For cases where you'd like to truncate to certain number of characters, a pipe would be the way to go.
Articles

The <article> tag surrounds copy in product documentation. This copy is often denser than what's ideal for application UIs. In limited instances, elements of product documentation require unique typography treatment.

Article lists

Formatting is provided for nested lists in <article> and <article><table> blocks using either the <ul> or <ol> tags. Lists in tables are more compact than lists in paragraphs. For example, the font-size is 1rem or 14px and margins and padding are tighter.

  • Item 2

Form Fields

Text formatting

For both the label on select boxes and their contents, please use the following formatting: Capitalize first word only, except in cases where the selection is an actual program name or proper name (e.g. Patient non-adherence or challenges in care management; Duplicate patient; Algorithm incorrect; Comprehensive HIV AIDS Management Program (CHAMP), Grace Program, LSU Diabetes, Salt Lake City, etc.).

Labels

Use a colon at the end of form field labels (e.g. Email address:). A colon implies a direct connection between the label text and a particular control or set of controls. The only situation where a colon should not be used is when the label and control work together to form a single sentence.

Hospital department:

Sass Text Utilities

Save yourself the trouble of looking up or recalculating rem sizes using the fontSize() sass mixin.

@import '../node_modules/cashmere/sass/cashmere';
.some-class {{ '{' }}
    @include fontSize(16px); // generates font-size: 1.143rem;
{{ '}' }}
