Favicon Guidelines

Last updated May 17, 2023
Overview

Favicons are the small icons displayed in the browser's address bar next to the page's title. They're also used for bookmarks. We have a standard look and feel for favicons using Cashmere $blue overlayed with either the app's initials or the Health Catalyst riflame.

The Health Catalyst design team has a template they can use to generate these icons, so we recommend reaching out to #design on slack to request one.
Favicon Generator

After obtaining the base favicon asset from the design team, we highly recommend using Real Favicon Generator to generate the icons. You'll provide it a 260x260 .png and it spits out the markup and files you need for rock solid favicons.

In addition to generating standard desktop web favicons, Favicon Generator provides everything needed to set up to have icons available for mobile platforms. So if a user bookmarks your application from a tablet, or creates a shortcut link on their phone's home screen, we'll have good looking icons at the ready.

Favicon Setup with Favicon Generator:
  1. Obtain a 260x260px png favicon from the design team.
  2. Go to Real Favicon Generator and click "select your favicon image".
  3. We'll leave most of the configuration options presented as is, but will edit the output path.
    1. Scroll down to the bottom of the configuration page to change the path. Favicon Generator Options > Path.
    2. Select "I cannot or do not want to place favicon files at the root...
    3. Create a new folder in your codebase under to house the files the generator will create. Something like src/assets/favico.
    4. Enter the path to your newly created favico folder.
  4. Click the Generate button.
  5. Download the generated .zip file and extract in the folder reference above. We do this to avoid muddying up the root directory.
    • Also add a copy of favico.ico (from the .zip file) at the root of your web app right next to index.html. We do this as a fallback.
  6. Grab the generated HTML and stick it in the $gt;head< of your index.html file.
  7. Might be worth linking to this process in your own webapp's readme in case future updates are needed to your favicon.
