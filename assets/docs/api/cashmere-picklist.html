PicklistComponent

Control for selecting multiple items from a very large set of options. <hc-picklist>

Properties
Name Description

add: EventEmitter<Array<string | Record<string, unknown> | undefined>>

Fires when options are added. Sends an array of the values being added.

addCustomItem: boolean | AddCustomItemFn

True if user should be able to add custom items. The button to add an option will appear when a search term is not exactly matched. Can optionally provide a function that will generate a value for the custom option based on the given search term. Defaults to false. If a function is provided, must follow this signature: ((term: string) => any | Promise<any>). The function is passed the search term string and returns a value that should match the type of options in the picklist.

addCustomItemText: 'Add custom option.'

When addCustomItem is true, this changes the default message for adding a new item. Defaults to "Add custom option."

bindLabel: string

If options are objects, this matches the object property to use for display in the list. Can use nested properties. Example: 'property.nestedProperty'. Defaults to 'label'.

bindValue: string

If options are objects, this matches the object property to use for the selected model. Can use nested properties. Example: 'property.nestedProperty'. By default, binds to whole object.

bufferAmount: 4

When using virtual scroll, determines how many extra items on top and bottom should be rendered outside the viewport. Defaults to 4.

canCloseGroup: false

True if group can be closed to hide its child options. Defaults to false.

canSelectGroup: false

True if group should be clickable. Clicking the group will select all its children. Defaults to false.

change: EventEmitter<Array<string | Record<string, unknown> | undefined>>

Fires when model is updated. Sends an array of the currently selected values.

closeGroupsByDefault: true

True if groups should be closed to begin with. Must have canCloseGroup set to true to use. Defaults to true.

compareWith: CompareWithFn

A function to compare the option values with the selected values. (a: any, b: any) => boolean; The first argument is a value from an option. The second is a value from the selection (model). A boolean should be returned. Same as used by https://angular.io/api/forms/SelectControlValueAccessor

disabled: boolean

Getter. Returns true if readonly property is true, or if disabled attribute is present on the control.

externalSearchSubject: Subject<string>

A subject through which an updated search term will be sent. Use for advanced searching functionality, like searching over http.

externalSearchTermMinLength: 0

Number of characters required for search to be fired. Only applies when using externalSearchSubject. Defaults to 0.

externalTotalOptionCount: number

For the left pane, force the 'Showing x of y' message to be the given number. Useful when using external search and you want to display the total number of items available on the server rather than just those currently available in the component.

groupBy: string | GroupByFn

Group items by property or function expression. If a function is provided, must follow this signature: ((item: any) => any). The function is passed an option and returns a value to represent the identifier for the group that option should belong to. Grouping is off by default.

groupValue: GroupValueFn

Function expression to provide group value: (key: string | object, children: any[]) => string | object. Same as described in ng-select component. https://ng-select.github.io/ng-select#/grouping

hasExceededMaxItemsSelected: boolean

Getter. Returns true if the number of selected items exceeds the maximum number. UI won't normally allow this, but the model can be manipulated or set outside of the component to cause this.

hasFooter: true

True if each pane should have a footer section. Defaults to true.

hasHeader: true

True if each pane should have a header. Defaults to true.

hasMaxItemsSelected: boolean

Getter. Returns true if the number of selected items matches or exceeds the maximum number. Useful if you'd like to show a custom warning label.

hasSearch: true

True if search bar should be present. Defaults to true.

hasToolbar: true

True if each pane should have a toolbar section. Defaults to true.

hasValue: boolean

Getter. Returns true if any items are selected.

height: '400px'

Total height of the picklist control. Should be passed as a string, including the unit (pixels, percentage, rem, etc). Defaults to '400px'.

items: unknown[]

An array of options for the picklist. Options can be of any type, and the array can be an observable stream. Can alternatively use <hc-pick-option> components to pass in options.

leftHeaderText: 'Available'

Text for left pane header. Ignored if hcPaneHeaderLeftTmp is used. Defaults to 'Available'.

leftPaneIsEmpty: boolean

Getter. Returns true if the "Available" pane is empty.

leftPaneLoading: false

True when left pane should show as loading. Defaults to false.

maxSelectedItems: number

Set a limit to the number of selected items. Note: This will not prevent more than limit from being added to the model outside the component. No limit by default.

notFoundText: 'No options to show.'

Message shown in each pane when empty. Defaults to 'No options to show.'

orphanItemsGroupName: 'Other Items'

Items that cannot be grouped are placed in an orphan group. This property sets a name for that group. Defaults to "Other Items"

readonly: false

If true, items can be viewed but not highlighted or moved from pane to pane. Same as adding disabled attribute. Defaults to false.

remove: EventEmitter<Array<string | Record<string, unknown> | undefined>>

Fires when options are removed. Sends an array of the values being removed.

rightHeaderText: 'Selected'

Text for right pane header. Ignored if hcPaneHeaderRightTmp is used. Defaults to 'Selected'.

rightPaneLoading: false

True when right pane should show as loading. Defaults to false.

searchFn: SearchFn

A custom search function: (term: string, item: any) => boolean. Default function does a simple scan for entire given term within the options label.

searchPlaceholder: 'Search'

Placeholder for the search input. Defaults to 'Search'.

searchWhileComposing: true

True if items should be filtered while user is still typing. Defaults to true.

sortFn: SortFn

Function used to sort the groups and items in the list: (a: PickOption, b: PickOption) => number. If the function result is negative, a is sorted before b. If it's positive, b is sorted before a. If the result is 0, no changes are done with the sort order of the two values. By default, options are not sorted.

trackByFn: null

Provide custom trackBy function for better DOM reusage by Angular's *ngFor in the items list.

virtualScroll: false

True to enable virtual scroll for better performance when rendering a lot of data. Defaults to false.
Methods
moveLeftToRight()

Move highlighted items from the left pane over to the right pane.
moveRightToLeft()

Move highlighted items from the right pane over to the left pane.
refreshScrollArea()

Refreshes the dimensions of the virtual scroll container and the items displayed within. Helpful when using virtual scrolling and the window changes such that the picklist was resized.
