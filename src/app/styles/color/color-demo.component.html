Health Catalyst UI Colors

Last updated Jul 8, 2019
Overview

Following are the suggested colors for use within software applications. Some expansion beyond the general palette is acceptable as long as the additional colors are similar in value/saturation. It's recommended that you import the colors and leverage the sass variables.
Primary Palette

These colors are used for the primary calls to action (buttons, links, etc) on a page.
Neutral Palette

The colors used for backgrounds, text, etc.
Secondary Palette

Additional options for accent colors.
General Color Variables

Some general-use color variables, referencing variables above, to decouple color name from an intended purpose.

  • $primary-brand - $blue
  • $primary-action - $green
  • $primary-alt-action - $purple
  • $destructive-action - $red
  • $neutral-action - $gray-400
  • $error - $deep-red
  • $text - $offblack
Stratifying and Adding Colors

There may be times that a product will need to expand the color palette, or use a variation on exisiting colors. In these cases we recommend one of two approaches:

  1. If importing the sass variables into your application, first try manipulating an existing color variable using the built in sass functions darken() or lighten(), or the library provided shade() or tint() functions. 
    @import "../colors";

.my-component {
    background-color: darken($slate-gray-100, 5%);
    color: tint($dark-blue, 5%);
}
  2. If that doesn't suffice, or you are not using the sass variables, try using coolors.co to help you stratify the base colors above or to generate new colors that fit within the color scheme. This color tool will also help you determine when you should use dark or light text on top of your color.
