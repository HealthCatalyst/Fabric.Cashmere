Icons v1 (Deprecated)

Last updated May 26, 2020
Overview
NOTE: The V1 icon set is now deprecated. Teams should migrate to the V2 icon set.

When symbols are needed, Health Catalyst makes use of two icon font families. First is Font Awesome, an industry standard for common UI iconography. And second is our custom HC Icons font, which acts as a complement to Font Awesome and provides a standard visual language around concepts frequently used in our applications.

The purpose of Health Catalyst’s icon set is to provide a standard visual language around concepts frequently used in our applications. For an optimal user experience, any time elements like data marts, bindings, or columns appear, they should be represented the same way – regardless of the application.

Note that Cashmere does not have a dependency on any icon font. Any glyphs that are used in the components have been base64 encoded into the component’s styles.

Usage

Different situations will call for different uses of these glyphs. The hc-icons font set is included with the Cashmere npm package. To include the v1 font in your application, add the following line to your global sass stylesheet:
@import "@healthcatalyst/cashmere/hcicons/hcicons";


If you're working on a project that isn't using sass, but can still pull the Cashmere npm package, a css file is also included:
node_modules/@healthcatalyst/cashmere/hcicons/css/hcicons.css


If you'd like to install the font on your computer or add it to your project manually, you may download the font files and stylesheets below.
