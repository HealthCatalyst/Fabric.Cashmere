Last updated Feb 1, 2022
Internet Explorer Support

Health Catalyst made the decision to drop support for Internet Explorer starting on June 15, 2022. This is in line with Microsoft's end-of-life date for the browser.

Because many healthcare organizations have not fully migrated to more modern browsers, we want to be sure to provide adequate warning within our applications. Example snippets are below, which you can adjust as needed for your application.

In releases of our applications before June 15th, 2022, if a user logs in using Internet Explorer, we want to provide a toast message explaining the upcoming change and linking to the HC Community announcement .

2. In app.component.ts, trigger the toast when IE is detected:


    @ViewChild('ieToastTemplate', { static: true }) ieToastTemplate: TemplateRef<any>;
    public ieToast;

    public ngOnInit(): void {
        this.checkForIE();
    }

    public closeIEToast(): void {
        this.ieToast.close();
    }

    private checkForIE() {
        if (/(MSIE|Trident\/)/i.test(navigator.userAgent)) {
            this.ieToast = this.toaster.addToast({
                position: 'bottom-right',
                timeout: 0,
                type: 'custom',
                width: 410
            }, this.ieToastTemplate);
        }
    }
After Dropping IE Support

In releases of our applications after June 15th, 2022, if a user logs in using Internet Explorer, we want to show a fullscreen message explaining that IE is no longer supported and they'll need to upgrade their browser.

1. In your angular.json, include the Cashmere scripts in your application's build assets:



    'build': {

        ...

        'options': {

            ...

            'assets': [
                // your other assets
                {
                    "glob": "**/*",
                    "input": "./node_modules/@healthcatalyst/cashmere/scripts/",
                    "output": "./scripts"
                }
            ]
        }
    }

2. In your index.html, before importing other scripts, reference the ie-unsupported.js script:


    <head>

        ...

        <!-- Show "No IE Support" message as needed. -->
        <script type='text/javascript' src='scripts/ie-unsupported.js'></script>

        ...

    </head>

Note: The path you use to reference this script may vary depending on your build system or architecture.
