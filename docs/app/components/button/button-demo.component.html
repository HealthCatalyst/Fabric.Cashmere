About

Buttons communicate the action that will occur when the user touches them. A Split Button is a composite control with which the user can choose a default action, or choose from a list of alternate actions displayed in a popover.

Color Variations

Above are all acceptable colors and their active states. Green should be used for most cases and especially cases the action is positive or in agreement. For negative actions use red. The purple and gray variations should be used only in cases where more colors are needed or if they are being placed on backgrounds which green doesn’t contrast well enough. Icons, if needed, should match text color of the button.

Button Usage

Generally, there should only be one primary button per page. All other interactive items should either be secondary or tertiary buttons. A few scenarios in which there would be multiple primary buttons: a page with both an “accept” and “decline” action, a page who’s content are non related and each content section have disparate actions associated with them. For modals the preference is one primary button and a cancel action set as a tertiary button.


A split button should be used when there is a single action most commonly used, but a series of alternate actions that a user may occassionally take. An example would be a "Save" button, which includes options such as "Save As" and "Save a Copy" in the split popover.


Button styles may be used either within <button> or <a> tags depending on which is best for the particular situation.
