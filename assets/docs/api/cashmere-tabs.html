Additional classes

TabComponent

Properties
Name Description

exactRouteMatch: boolean

If set to true, [routerLinkActiveOptions]="{exact: true}" is added

hidden: boolean

Overrides the default Angular hidden behavior to support overflow states; ngIf is also supported to hide tabs

maxWidth: 'none'

Tab widths default to fit content; use maxWidth to constrain and truncate the content to a fixed width (e.g. 150px)

queryParams: Params

Add queryParams to the routerLink

routerLink: string[] | string

Router path that the tab routes to. If one tab uses the routerLink in a tab set, all must use the router link. Can be specified as '/path/2' or ['path', '2']

tabClick: EventEmitter<Event>

Emits when this tab is selected; use instead of (click) for click binding

tabContent: TemplateRef<unknown>

The template to be used when this tab is selected. Defaults to the content of this tab component. Not used when the tab set uses routing.

tabTitle: ''

Plain text title of the tab; for HTML support include a hc-tab-title element

TabSetComponent

Properties
Name Description

addContentContainer: boolean

Determines whether the tab set will create a router-outlet or ng-container for the tab content. If set to false, the app will need to add its own container. Defaults to true.

defaultTab: string | number

Zero-based numerical value specifying which tab to select by default, setting to none means no tab will be immediately selected. Defaults to 0 (the first tab). For tabs using routing, the default tab will be set by the url and use this value as a fallback if no tab routerLinks match the url. NOTE - if using the selectedTab param, it will override this value.

direction: string

Specify direction of tabs as either horizontal or vertical. Defaults to vertical

overflowStyle: string

When horzontal tabs overflow the container, specify either 'more', 'arrows', or 'none' for navigation control. Defaults to more

selectedTab: number | TabComponent

Write-only*: specify which tab is currently selected. Does not fire a selectedTabChange emission.

selectedTabChange: EventEmitter<TabChangeEvent>

Emits when the selected tab is changed. Use to keep track of the currently selected tab.

tabContent: TemplateRef<unknown> | null

The content to be displayed for the currently selected tab. This is read from the tab when it is selected. Not used when this component uses routing.

tight: boolean

If true, condense the default padding on all included tabs. Defaults to false.
Methods
refreshTabWidths(forceRecalulate: boolean = false)

Forces a recalculation of the tabs to determine how many should be rolling into a More menu. By default the function will use cached values of the tab widths; if you've changed the title of a tab, set forceRecalculate to true to update the cache
selectTab(tab: number | TabComponent, shouldEmit: boolean = true)

Sets the currently selected tab by either its numerical index or TabComponent object. Passing a value of -1 or a hidden tab will deselect all tabs in the set.

TabChangeEvent

Object returned by a selectedTabChange event; a -1 index is returned if all tabs are deselected
