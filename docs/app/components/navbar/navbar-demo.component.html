Responsive Behavior

The navbar is fully responsive and collapses in series of steps. At desktop width, the navbar will check to see if all hc-navbar-links can fit on the screen. If not, it will reduce the right-side padding on the logo. If there still isn't enough space, it will begin collapsing hc-navbar-links into a "More" menu. Once the navbar reaches its first responsive breakpoint (tablet size) it will remove all hc-navbar-links and hc-navbar-icons and insert them into a mobile menu. At the second breakpoint (phone size) the hc-app-switcher is removed as well, leaving only the logo and the mobile menu.
